# RC1 Desktop Studio Checklist

Diese Checkliste entscheidet, ob eine Version als RC1-fähig gilt.

## 1. Mobile Schutztest

Vor und nach jedem Desktop-Patch prüfen:

- Mobile öffnet Szene korrekt.
- Mobile öffnet Text/Werbetext mit Einstieg.
- Mobile Textfelder sind zuverlässig editierbar.
- Texthöhe funktioniert im Bereich 4–88%.
- Mobile Buttons: Text, Aktion, Look, Verwalten funktionieren.
- Mobile Weblink einfügen funktioniert.
- Spot neu starten funktioniert.
- Mobile Public zeigt gleiche Größen wie Editor.

## 2. Desktop Hauptnavigation

Alle vier Hauptbereiche sichtbar und bedienbar:

- Szene
- Text
- Buttons
- Meine Ureel Seite

Jeder Wechsel aktualisiert die Mittelspalte und den rechten Editor.

## 3. Desktop Mittelspalte

Für alle Auswahlkarten:

- Titel und Beschreibung getrennt.
- Keine abgeschnittenen Wörter.
- Aktive Auswahl klar markiert.
- Keine reine Textzeile statt Button.
- Einheitliche Abstände und Höhen.

## 4. Rechter Editor

Jeder Bereich zeigt den richtigen aktiven Editor:

- Szene → Video/Bild/Farbe/Darstellung/Endkarte/Profilbild.
- Text → Werbetext/Vorlagen/Rahmen & Stil/Timing.
- Buttons → Text/Aktion/Look/Verwalten.
- Meine Ureel Seite → oben Website-Vorschau, darunter aktiver Miniwebseiten-Editor.

## 5. Meine Ureel Seite

- Vorschau zeigt alle drei Bereiche sauber.
- Bereich 1: Karte positionierbar.
- Bereich 2: Begrüßung und Button-Menü.
- Bereich 3: Text und Bild/Video.
- Webseite starten: Öffnen, Kopieren, Teilen, QR, Kontakt prüfen.
- Reale Desktop-Webseite zeigt keine internen Labels wie „Bereich 1/2/3“.

## 6. Public View

- Mobile Public bleibt 9:16.
- Desktop Public zeigt Miniwebseite, wenn aktiviert.
- Buttons sind klickbar.
- Video/Poster zeigt keinen leeren schwarzen Startzustand, soweit technisch möglich.

## 7. Browser

Manuell testen:

- Chrome Desktop
- Safari Desktop
- Edge Desktop
- Firefox Desktop
- iPhone Safari
- Android Chrome

## 8. Freigabe

Eine Version gilt erst als RC1-fähig, wenn alle Punkte ohne kritische Fehler durchlaufen sind.
